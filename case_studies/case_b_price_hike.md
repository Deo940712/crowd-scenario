# CASE B — 產品漲價，價值提升不足

> **這是情境排練，不是預測。** 用合成使用者原型排練「一個 SaaS 產品漲價、但新版感知
> 價值不升反降」的社群反應。輸入是**手工整理的原型情境**，不是任何真實產品的數據；引擎
> 不預測留存/流失，只演「若使用者長這樣、會怎麼反應」。重現：`python case_studies/report.py B`。

## 1. 情境背景

一個 SaaS 產品**明顯漲價**，但這一版帶來的**感知價值反而變差**，且使用者**切換到競品的
成本中等**（不是完全綁死）。這是 PM 最怕的組合。我們問：「**如果**一群典型使用者原型看到
這個組合，整體氣氛會怎樣？和 PM 自己的樂觀判讀差多少？」

## 2. 輸入：原始數字 → buckets（數字在這裡消失）

| 原始指標（只在這裡出現） | 值 | → bucket |
| --- | --- | --- |
| price_change | 0.3 | `steep_hike` |
| value_delta | -0.3 | `worse` |
| switching_cost | 0.6 | `medium` |

原始 `0.3 / -0.3 / 0.6` 到此為止，不進入任何 persona 語音或敘事。

## 3. Personas 反應（default = aggregate_neutral）

stance 分布：**-1 八人 / 0 零人 / +1 零人**——**全體反對，無人支持**。

| stance | persona | 代表語音 |
| --- | --- | --- |
| -1 | 重度用戶 | 這種漲法讓人卻步，考慮回頭找舊的替代品。 |
| -1 | 免費用戶 | 值得的好用戶才會付，否則直接放棄。 |
| -1 | 精打細算族 | 漲價又沒新價值，這種最容易被比下去。 |
| -1 | 品牌鐵粉 | 感情上支持，但也考慮切換成本值不值得。 |
| -1 | 競品愛好者 | 正合我意，趁機推坑大家看別家。 |
| -1 | 社群風向 | 論壇一面倒罵，帶頭發起討論。 |
| -1 | 嘗鮮玩家 | 沒新意就不玩了，起身就走。 |
| -1 | 輕度用戶 | 切換成本自己扛得住，回頭再想。 |

**八種型態一致反對**——當「漲價＋價值變差」兩個負向軸疊加、切換成本又不高，群眾方向
非常明確。

## 4. 三種 consensus 模式對照（本案例的重點）

| 模式 | 共識 | 說明 |
| --- | --- | --- |
| `aggregate_neutral`（default） | **negative** | 由人格多數決（-8 淨值），跟隨情境 |
| `hashed` | **neutral** | 種子雜湊擲出中性——**看不出這個明確負向情境** |
| `aggregate` | negative | 人格多數決 |

**這是 default 遷移（0.2.0）的活教材**：hashed 對這個「八人全空」的明確情境竟擲出
`neutral`，而 default 的 `aggregate_neutral` 正確反映 `negative`。這就是為什麼共識方向
現在跟隨情境語義、而不是骰子。

## 6. Divergence：群眾 vs 你自己的判讀

假設**你是 PM**，判讀 `positive`（相信漲價沒問題）。群眾 `negative`：

- divergence bucket：**HIGH**（方向相反）
- 反事實 storylines（條件式，無數字）：
  - 合成群眾偏保守，而你的判讀偏樂觀，兩者顯著相反。
  - 若群眾這一側較貼近後續，你這類判讀可能忽略了群體會如何反應（條件式推演）。
  - 若你這一側較貼近，群眾這類反應可能是在追逐雜訊而非訊號（條件式推演）。
  - 顯著分歧本身就是值得留意的排練訊號，但它不決定任何動作。

## 7. 哪裡有用（What this told us）

- **HIGH divergence 是一記警鐘**：PM 的樂觀與群眾的一致反對完全相反，反事實劇本逼你
  正面回答「我是不是低估了『漲價＋無價值』的反彈？」
- **反對是『全面且理由分歧』的**：重度用戶談替代品、精打細算族談比價、社群風向談帶頭
  罵——不是單一原因，這對「該補價值還是收回漲價」的討論很有料。
- **模式對照揭露了骰子問題**：同一情境 hashed=neutral、default=negative，示範了為何
  0.2.0 把 default 改成跟隨情境。

## 8. 哪裡不能信（What this CANNOT tell us）

- **它不預測真實流失率或營收影響。** negative 是合成群眾傾向，不是留存預測，未經回測。
- **八種 cohort 是刻板原型**，不含你的真實用戶結構、定價彈性、市場競態。
- **輸入是手工原型情境**；真實的漲幅/價值/切換成本一變，桶與結論都可能不同。
- **『群眾一致反對』不等於漲價一定失敗**：群眾也可能過度反應，或你的產品有模型看不到的
  黏著。這是排練警示，不是決策。
